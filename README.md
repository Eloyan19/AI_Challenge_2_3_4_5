# AI Challenge — Android DeepSeek Agent

Android-приложение на Kotlin + Jetpack Compose, реализующее LLM-агента с инструментами поверх [DeepSeek API](https://platform.deepseek.com/).

## Что такое агент в этом приложении

Агент — это не просто обёртка над API. Это отдельная сущность (`SimpleAgent`), которая:

- **помнит контекст** — хранит историю сообщений и передаёт её в каждом запросе, позволяя вести полноценный диалог
- **инкапсулирует логику** — построение запроса, расчёт стоимости, обработка ошибок спрятаны внутри агента
- **умеет действовать** — сам решает когда и какой инструмент вызвать, выполняет его и возвращает результат в LLM

```
SimpleAgent
├── run(userInput)     — принимает сообщение пользователя, запускает цикл
├── agentLoop()        — LLM → tool call? → execute → LLM → ... → ответ
├── buildRequest()     — собирает ChatRequest с историей и схемами инструментов
├── calculateCost()    — считает стоимость по токенам
└── reset()            — очищает память (новая сессия)
```

## Агентный цикл (Agentic Loop)

```
Пользователь: "Какая погода в Ереване и сколько стоит 100 EUR в рублях?"
       ↓
  LLM думает: нужны инструменты
       ↓
  get_weather("Ереван")  +  convert_currency(100, "EUR", "RUB")
       ↓
  ToolExecutor выполняет запросы к внешним API
       ↓
  Результаты передаются обратно в LLM
       ↓
  LLM формирует финальный ответ пользователю
```

LLM сама решает, нужен ли инструмент, какой и с какими аргументами — это и есть агентность.

## Инструменты

| Инструмент | Что делает | API | Ключ |
|---|---|---|---|
| `get_weather` | Текущая погода для любого города | [wttr.in](https://wttr.in/) | не нужен |
| `convert_currency` | Конвертация валют по актуальному курсу | [Frankfurter.app](https://www.frankfurter.app/) | не нужен |
| `web_search` | Поиск актуальной информации в интернете | [Yandex XML Search](https://xml.yandex.com/) | нужен (бесплатный, 1000 запросов/день) |

Инструменты автоматически отключаются в Thinking Mode, так как DeepSeek-R1 не поддерживает tool calls.

## Архитектура

```
UI (MainActivity)
    └── MainViewModel                — состояние UI, SessionStats, обнаружение overflow
            └── SimpleAgent          — ядро: память + LLM-логика + цикл инструментов
                    ├── ContextCompressor             — сжатие контекста + LLM-суммаризация
                    ├── DeepSeekApiService (Retrofit) — запросы к DeepSeek
                    ├── ToolDefinitions               — JSON-схемы инструментов
                    └── ToolExecutor                  — выполнение инструментов
                            ├── wttr.in               — погода
                            ├── Frankfurter API       — валюты
                            └── Yandex XML Search API — поиск

Room DB (version 2)
    ├── chat_messages       — история сообщений (по turn_id)
    └── conversation_summary — текущий summary сжатого контекста
```

## Функции

- Чат-интерфейс с историей диалога
- Поддержка моделей `deepseek-v4-flash` и `deepseek-v4-pro`
- Thinking Mode с выбором Reasoning Effort
- Инструменты: погода, конвертер валют, поиск в интернете
- Статус активного инструмента отображается во время загрузки
- Персистентность истории и summary через Room (восстанавливается после перезапуска)
- Кнопка «Новая сессия» — сбрасывает память агента, историю и summary
- **Сжатие контекста** — хранение последних N сообщений + LLM-generated summary

### Статистика токенов и контекста

Под каждым ответом агента отображается разбивка:
```
↑ 1.2K (кэш 800) • ↓ 256 • $0.000123 • 2.3 с
```
- `↑` — токены запроса (prompt), в скобках — кэшированные
- `↓` — токены ответа (completion)

Панель статистики сессии над полем ввода показывает накопленные данные и заполненность контекста (лимит модели — 128K токенов):

```
Ходов: 5  |  Сгенерировано: 3.2K tok  |  Потрачено: $0.00045
Контекст  [████████░░░░░░░░░░░]  42% · 54.2K/128.0K
```

Цвет прогресс-бара меняется по мере роста контекста:
- 🟢 < 70% — в норме
- 🟠 70–85% — приближается к пределу
- 🔴 > 85% — предупреждение; при ошибке показывается подсказка «Начните новую сессию»

### Сжатие контекста (Context Compression)

Фича позволяет сравнить расход токенов и качество ответов с компрессией и без.

**Включение:** «Дополнительно» → переключатель «Сжатие контекста».

**Принцип работы:**
1. Агент хранит последние **N** сообщений в полном виде (настраивается).
2. При превышении порога старые сообщения заменяются **summary** — коротким пересказом (3–6 предложений), сгенерированным через `deepseek-v4-flash`.
3. Summary подставляется в каждый запрос как system-сообщение, чтобы модель помнила контекст.
4. Summary персистируется в Room и восстанавливается при перезапуске приложения.

```
SimpleAgent
├── run(userInput)
│       └── ContextCompressor.prepareContext(history)  ← сжимает при необходимости
├── buildRequest()
│       └── ContextCompressor.buildMessages(history)   ← prepend summary
└── ContextCompressor
        ├── prepareContext()   — суммаризирует excess-сообщения через LLM
        ├── buildMessages()    — вставляет summary перед историей
        └── generateSummary()  — вызов deepseek-v4-flash, maxTokens=600
```

Статус отображается в панели над полем ввода:
```
🗜 Сжатие ON · последние 10 сообщений · summary: есть ✓
Пользователь спрашивал о погоде в Барселоне и конвертации валют…
```

Фича полностью выключаемая — при отключении агент снова использует полную историю.

## Настройка

Добавь ключи в `local.properties`:

```properties
DEEPSEEK_API_KEY=sk-...

# Опционально — для инструмента web_search
# Ключ получить на https://xml.yandex.com/ (бесплатно, 1000 запросов/день)
YANDEX_SEARCH_USER=логин_яндекса
YANDEX_SEARCH_KEY=ключ_из_xml.yandex.com
```
